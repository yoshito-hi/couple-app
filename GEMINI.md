# プロジェクトの基本情報

## 1. プロジェクト概要

- **目的**: 友人との思い出を年表や四コマ漫画として記録・振り返るための専用アプリケーション。
- **ターゲット/規模**: 開発者含む友人2名のみのクローズド利用。
- **運用コスト**: トラフィックが限定的なため、フルマネージドサービスの**完全無料枠（Hobbyプラン等）**の範囲内で運用する。

## 2. 画面構成（想定）

- **ログイン画面**: ユーザー認証用。ログインユーザーに応じてアプリ全体のカラーテーマを動的に変更する。
- **思い出年表画面**: 過去の出来事を時系列で表示するタイムライン。
- **思い出漫画閲覧画面**: 個別の四コマ漫画を拡大して閲覧する詳細ページ。

## 3. 技術スタック・インフラ構成

| カテゴリ           | 採用技術・サービス                                | 備考・用途                                                          |
| :----------------- | :------------------------------------------------ | :------------------------------------------------------------------ |
| フロントエンド     | Next.js (App Router), React、shadcn/ui, storybook | UI構築、クライアントサイドのルーティング                            |
| バックエンド (BFF) | Next.js (App Router)                              | Server Components / Server Actions を用いたデータ処理               |
| 言語               | TypeScript                                        | 型安全な開発の担保                                                  |
| BaaS               | Supabase                                          | バックエンド基盤（DB、認証、ストレージ）                            |
| データベース       | PostgreSQL (Supabase内蔵)                         | アプリケーションデータの保存。RLS(行単位セキュリティ)でアクセス制御 |
| ファイルストレージ | Supabase Storage                                  | 漫画の画像ファイルや静的アセットの保存                              |
| ホスティング       | Vercel                                            | Next.jsアプリケーションの実行環境                                   |

## 4. 認証・セキュリティ

- **認証プロバイダ**: Supabase Auth
  - 2名限定の利用のため、招待制（Magic Link等）やEmail/Passwordベースでシンプルに構築。
- **データアクセス制御**:
  - Next.js側からのアクセスには `@supabase/ssr` を使用し、Cookieベースのセッション管理を実施。
  - Supabaseの **RLS (Row Level Security)** を有効化し、認証済みユーザーのみがデータや画像にアクセスできるよう制限する。

## 5. データモデルの考え方

利用者が2名限定であるため、ユーザー属性（表示名、テーマカラー、役割など）はデータベース（profilesテーブル等）で管理せず、フロントエンド側の設定（定数）として保持します。これによりSupabase側の構成をシンプルに保ちます。

- **events テーブル**: 年表用の出来事データ（日付、タイトル、詳細）。
- **comics テーブル**: 漫画のメタデータ（タイトル、投稿日、Storage内の画像パス）。

## 6. CI/CD・開発体験（DX）

- **デプロイフロー**: Vercel + GitHub連携
  - GitHubの `main` (または `production`) ブランチへのPush / Mergeをトリガーに、Vercelが自動でビルド＆デプロイを実行。
- **Linter / Formatter**: Biome
  - Prettier/ESLintの代替として採用。超高速なコードフォーマットと静的解析で開発体験を向上。
- **バリデーション / 型定義**: Zod
  - フォーム入力やAPI/DBから取得したデータのスキーマ定義・バリデーションに使用し、TypeScriptの型と同期させる。
